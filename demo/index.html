A complete walkthrough from raw data analysis to model recommendation — covering null treatment, univariate & bivariate analysis, feature engineering, and intelligent model selection based on dataset characteristics.

Using built-in e-commerce dataset
🎯 Problem Statement
What are we predicting?
Predict the per-order revenue for each e-commerce transaction, where revenue = price × quantity × (1 − discount%). The model learns how product category (Electronics vs Books vs Clothing etc.), unit price, quantity ordered, discount applied, customer segment, and region together determine how much revenue a single order generates.
📈 Key drivers of revenue
• Category — Electronics orders average 5× more revenue than Books
• Price — strongest linear predictor (log-transformed)
• Quantity — each extra unit adds proportional revenue
• Discount — higher discount directly reduces revenue
📌 Why predict per-order revenue?
• Identify which product + region combinations are most profitable
• Flag high-value orders before fulfilment for priority handling
• Optimise discount strategy — avoid over-discounting Electronics
• Forecast total revenue from incoming order pipelines

Multivariate Analysis — Correlation Heatmap

👁 HOW TO READ THE HEATMAP
Dark Red — r near +1.0
Strong positive relationship. As one feature rises, the other rises too.
Light Red — r +0.4 to +0.7
Moderate positive trend. Some shared information between the two features.
White — r near 0
No linear relationship. The two features move independently of each other.
Light Blue — r −0.4 to −0.7
Moderate negative trend. As one rises, the other tends to fall.
Dark Blue — r near −1.0
Strong negative relationship. Features move in exactly opposite directions.
Outlined diagonal — r = 1.00
Every feature is perfectly correlated with itself. Ignore the diagonal.
🎯 HOW TO USE THIS FOR FEATURE SELECTION
1
Scan for dark red / dark blue cells — any cell with |r| ≥ 0.7 off the diagonal signals multicollinearity. Those two features carry nearly the same information.
2
Keep the feature more correlated with your target — if price and revenue both correlate with each other, keep whichever has a stronger relationship to what you are predicting.
3
Drop or combine the redundant one — dropping reduces noise. Combining (e.g. ratio or difference) can preserve information from both.
4
White cells = safe to keep both — features near r = 0 are independent and each adds unique signal to the model.
5
Confirm with VIF after dropping — Variance Inflation Factor catches multi-way collinearity that pairwise r can miss. Target VIF < 5 for all remaining features.

XGBoost Regressor
Recommended for this dataset
📊 WHY XGBoost IS RECOMMENDED FOR THIS DATASET
Dataset Profile 1,000 rows × 12 columns, mixed numerical + categorical features, right-skewed target (revenue), multicollinearity detected
Handles Skewness Tree-based splits naturally handle right-skewed price/revenue data without requiring log transforms
Mixed Feature Types Works with both numerical and categorical features natively — no separate encoding/scaling needed
Robust to Issues Not affected by multicollinearity between price and revenue, handles outliers better than linear models
Performance Typically achieves R² > 0.85 on similar e-commerce datasets with proper tuning
Alternatives Random Forest (interpretable, no tuning needed) • Ridge Regression (fast baseline, requires scaling)

💡 Implementation Notes — how to train XGBoost Regressor
After running this analysis, you now have a clean, processed dataset ready for modeling. Here's how to implement XGBoost Regressor:
📋 Training Steps
1. Use the processed data from Feature Engineering tab
2. Split 80/20 for train/validation
3. Target: revenue (continuous regression)
4. Features: all except order_id, order_date
5. Metric: RMSE or R² for evaluation
🔧 Key Hyperparameters
• n_estimators: 100-500
• max_depth: 3-6
• learning_rate: 0.1-0.3
• objective: 'reg:squarederror'
• eval_metric: 'rmse'
• early_stopping_rounds: 10
🎯 Expected Performance
With proper feature engineering (completed above), XGBoost Regressor should achieve R² > 0.80 with proper tuning. The strongest predictors will likely be: price → quantity → discount → category.
